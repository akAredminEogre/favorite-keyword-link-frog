# 解説
## 今回の設定値の肝
- wxt.config.tsのdevプロパティとviteプロパティのhostname、portは一致させる必要がある。
- またhostnameの値はdocker-compose.ymlのenvironmentプロパティのHOSTにも一致させる必要がある。
  - コンテナ内のViteサーバーとホストマシンのChrome拡張機能を通信できるようにするため。
- 現時点では、docker-compose.ymlのnetwork_modeはhostにする必要がある。(後述)



## vite.config.tsの内容はwxt.config.tsに記述する
公式ドキュメント
https://wxt.dev/guide/essentials/config/vite.html#change-vite-config
によれば、通常vite.config.tsのdefineConfigに記述する内容を、wxt.config.tsに記述することで、Viteの設定を変更できる。

```
// wxt.config.ts
import { defineConfig } from 'wxt';

export default defineConfig({
  vite: () => ({
    // Override config here, same as `defineConfig({ ... })`
    // inside vite.config.ts files
  }),
});
```

## network_mode: host について
本来は、docker-compose.ymlのnetwork_modeはhostにしないほうが良いが、WebSocket関連のエラーをうまく回避できなかったため、hostにしている。


# 未解決事項
## Dockerfileにおけるレイヤーキャッシュの利用

ビルド速度向上のために、ホスト側からのソースコードコピーの前に、npm installを実行することが望ましい。

```
# 1. package.json と lockファイルのみコピー
COPY ./host-frontend-root/frontend-src-root/package*.json ./

# 2. 先に依存関係をインストール
RUN npm install

# 3. 全ソースコードをコピー
COPY ./host-frontend-root/frontend-src-root ./
```
しかし、wxtの場合、npm installを実行すると、`npx wxt@latest init`コマンドで、デフォルトで用意されている
'''package.json
  "postinstall": "wxt prepare"
'''
が実行される。

しかし実際には、
```
ERROR  No entrypoints found in /opt/frontend-container-app-root/frontend-src-root/entrypoints
```
のエラーが発生してしまう。
つまり、 ` RUN npm install` の前にソースコードが用意されている必要がある。

どうしても必要であれば、
`package.json`から、`postinstall`を削除して、
Dockerfileを下記のように書き換えてリビルド、再実行。
```
# 1. package.json と lockファイルのみコピー
COPY ./host-frontend-root/frontend-src-root/package*.json ./

# 2. 先に依存関係をインストール (キャッシュが効きやすい)
RUN npm install

# 3. 全ソースコードをコピー
COPY ./host-frontend-root/frontend-src-root ./

# postinstallで指定されているコマンドを実行
RUN npx wxt prepare
```
これにより、レイヤーキャッシュを利用することはできるが、デフォルトで用意されている`postinstall`を削除することには抵抗があり、未解決とした。

## wxt.config.tsのrunner: { disabled: true } について
wxt.config.ts で、
```typescript:wxt.config.ts
  runner: {
    disabled: true,
  },
```
としているが、本当はfalse、できればこのオプションを消したい。しかし、現時点では、このオプションを消すと、下記のエラーが発生する。

```
  at TCPConnectWrap.afterConnect [as oncomplete] (node:net:1608:16)
```
公式ドキュメントでchromiumPort等の設定値も調べ試したが、解決できなかった。
一旦断念し、runnerを無効にし、手動で読み込むようにしている。
